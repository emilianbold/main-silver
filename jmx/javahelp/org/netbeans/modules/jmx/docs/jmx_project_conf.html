Java Application Project Configuration

Management actions are applied to the Java Application Project type only.

Configuration steps :
  1. You must set a Java Application project as your main projet.
  2. Correctly set your Runnable Class in the project properties. This can be done automatically when generating a JMX Agent.
  3. Check existing management related VM Options (if any). Be sure to remove these options from the project properties menu. They can conflict with these actions.
  4. FAT File System issue. Run Main Project with JConsole and Debug Main Project with JConsole actions do not work on FAT (see JConsole FAQ for more details on the issue http://java.sun.com/j2se/1.5.0/docs/guide/management/faq.html)
    • Use Run > Remote Management > Run Main Project with Remote Management or Debug Main Project with Remote Management actions instead
    • Or, to work around this issue, add -XX:+PerfBypassFileSystemCheck to the project's VM Options (in the project's Properties > Run > VM Options textfield). DO NOT use this option in a real environment. It is an unsupported option that works around the problem BUT that makes the management access unsecure.
