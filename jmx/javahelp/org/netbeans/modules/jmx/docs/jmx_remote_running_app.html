Running Your Project with Remote Management

  1. Click on Run/Remote Management/Run Main Project with Remote Management...
  2. If this is the first time you are running a management action, a dialog box pops up notifying you that the build file will be updated. Click Accept.
  3. A dialog box is displayed prompting you for some information. You can:
    1. Provide an RMI port. A JMXConnectorServer will wait on this port for incomming JMX requests.
    2. Or provide a management property file that will be used to set up a detailed configuration (RMI port, SNMP access and security)
    3. You can auto connect JConsole to your process by checking the appropriate checkbox
  4. Click OK
  5. Your process is compiled and run like any Java application. If JConsole was enabled, JConsole is displayed. Click on the MBeans Tab to visualize your MBeans.
  6. If JConsole is launched, close JConsole to stop the session.

Debugging Your Project with Remote Management

  1. Click on Run/Remote Management/Debug Main Project with Remote Management...
  2. If this is the first time you are running a management action, a dialog box pops up notifying you that the build file will be updated. Click Accept.
  3. A dialog box is displayed prompting you for some information. You can
    1. Provide an RMI port. A JMXConnectorServer will wait on this port for incomming JMX requests.
    2. Or provide a management property file that will be used to set up a detailed configuration (RMI port, SNMP access and security)
    3. You can auto connect JConsole to your process by clicking on the appropriate checkbox
  4. Click OK
  5. Your process is compiled and run like any Java application. The debugger is started. If JConsole was enabled, JConsole is displayed. Click on the MBeans Tab to visualize your MBeans
  6. If JConsole is launched, close JConsole to stop the process execution.
  7. Stop the debug Session.
