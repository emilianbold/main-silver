Generating an MBean from an Existing Java Class

The wizard allows you to generate an MBean based on an existing Java class.
The provided class is introspected in order to select what should be exported in the management interface.
The MBean that is generated is an extended Standard MBean.

Extended Standard MBean

An Extended Standard MBean is an MBean that extends the javax.management.StandardMBean class (see the JMX javadoc for more details on the StandardMBean class).
In the case of an MBean generated from a Java class, the generated MBean implementation delegates all its calls to an instance of the introspected class. A reference to an instance of the introispected class can be passed to the generated MBean constructor.

Using the MBean Wizard to Generate an MBean from an Existing Java Class

The following sections dissect an example standard MBean. The MBean wizard is used to generate the classes.

  1. Select File/New File...
  2. Select the JMX category. In File Types select JMX MBean From Existing Java Class
  3. Type in a Name, a Package
  4. Click Next>
  5. Check the From Existing Class checkbox.
  6. Click on the Browse... button in order to select a class from the current project. You can also enter your class name (package + name) in the textfield.
  7. You can enter a Description in order to document your MBean class. Any documentation that is typed from this wizard will be generated as :
    • Javadoc for Standard MBeans.
    • Added to the MBeanInfo in order to provide documentation on the MBean at runtime for Dynamic and Extended MBeans.
  8. Click Next>
  9. A set of discovered potential attributes (if any) are displayed in the attributes table. You can update the Attribute Description in order to provide interesting information on each attribute.
  10. You can filter out some attributes by unselecting the corresponding checkbox.
  11. You can also add new attributes. Click Add Attribute.
  12. In the Attributes Table, first row, type in the following:
    • In the column Name type in an Attribute Name (eg : Status)
    • In the column Type select a Java type (eg: String)
    • In the column Access select an access type (eg:Read / Write)
    • In the column Description, type in what this attribute represents (eg: This MBean's current Status)
  13. Click Next>
  14. A set of discovered operations (if any) are displayed in the attributes table. You can update the Operation Description in order to provide interesting information on each operation.
  15. You can filter out some operations by unselecting the corresponding checkbox.
  16. If you want your MBean to have new operations, click Add Operation.
  17. In the Operations Table, first row, type in the following:
    • In the column Name type in an operation name (eg : reset)
    • In the column Return Type select a Java type (eg: void)
    • If you want to add input parameters, in the column Parameters, click the Edit button.
      • Click Add Parameter
      • In the column Parameter Name you can change the default parameter name.
      • In the column Parameter Type, select a Java type (eg: String)
      • In the column Parameter Description  type in what this parameter represents.
    • In the column Description, type in what this operation represents (eg: Reset the current Status)
  18. Click Finish.
  19. The MBean implementation class is opened in the editor.  If you selected a Standard MBean type, an MBean interface (named <MBean Name>MBean), an implementation class (named <MBean Name>) and a JUnit test are generated in the src and test packages you selected.
  20. You can now add your own logic to the generated code and run the JUnit test.
