Implementing the javax.management.MBeanRegistration Interface


MBean registration is performed using a contextual action (enabled only if an MBean is selected) located under the Management menu. You can access the management menu from :
  • The contextual menu of the project tree node (right click on selected node)

Applicability of Actions


This action  applies to any MBean type. If an MBean already implements this interface, the action is disabled.
This action applies to any MBean. You don't need to have generated them using the MBean wizard.
MBeans imported from other contexts can be updated.

Using the Implement MBeanRegistration Interface Action


  1. From the JMX menu, select Implement MBeanRegistration Interface...
  2. A window is displayed with a message listing the set of methods that will be added to the current MBean
    • preRegister : Called before the MBean is registered in the MBeanServer
    • postRegister : Called after the MBean is registered in the MBeanServer
    • preDeregister : Called before the MBean is unregistered in the MBeanServer
    • postDeregister : Called after the MBean is unregistered in the MBeanServer
  3. You can generate private fields to keep references to the preRegister method parameters. These parameters are :
    • The MBean ObjectName. If you are interested in the name the MBean is given, you should keep a reference to this parameter.
    • The MBeanServer in which the MBean is registered. If you want your MBean to make JMX call to other MBeans, you should keep this reference.
  4. Click OK when you have finished
  5. MBeanRegistration methods are added at the end of the MBean class.
  6. The updated MBean class file is automatically opened in the editor.
