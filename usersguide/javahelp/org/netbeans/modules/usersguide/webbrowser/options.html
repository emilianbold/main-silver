Options Window: Web Browsers Manager

See Also

You can use the Web Browsers Manager to specify options for the web browsers that are registered with the IDE. You can also use this window to add and remove browsers from the list of registered browsers.

You access the Web Browsers Manager by clicking Edit next to the Web Browsers dropdown list in the General category of the Options window.

To add a web browser to the list of registered browsers, click Add below the Web Browsers list and specify the properties of the new web browser. To remove a web browser, select the browser in the list and click Remove.

To configure a web browser, select a browser from the list and modify the browser properties.

Properties

The Web Browsers Manager contains the following options for modifying web browser properties.

  • Web Browsers. This pane displays a list of web browsers on your local system and registered with the IDE. Select a browser from the list to edit the browser properties.

    In addition to the installed browsers, the list contains the following browsers by default.

    • <Default System Browser>. This option represents the default browser of your local system.
    • Embedded WebKit Browser. This option is a web browser that is based on the WebKit browser engine and bundled with the IDE.
    • External Browser. This option is a command-line browser.
  • Name. The display name for the browser that is used by the IDE.
  • Process. The full path to the browser executable. Click Browse to locate the executable on the local system.
  • Arguments. Arguments to pass to the executable when the browser is run.
  • If you select Embedded WebKit Browser in the list you can modify the following properties.
    • Use Another Browser for Web Application Development. When selected you can choose a web browser from the dropdown list to use as the default browser when you run web applications.
    • JavaFX Runtime Location. Click Browse to specify the location of the local JavaFX runtime. JavaFX is required to run the Embedded WebKit Browser. You might need to explicitly specify the JavaFX runtime location if your runtime is not installed in the default location or the Embedded WebKit Browser fails to run.
