Setting Up Security for Your Project

The Java Card 3 platform uses Secure Socket Layer (SSL) to provide security for applications by encrypting and decrypting data. An SSL implementation needs four algorithms, digital signature, key establishment, bulk encryption, and message digest.

The Java Card 3 platform implements the SSL key establishment algorithm through the use of a set of certificates and keys as key=value pairs specified in the config.properties file.

You do not have to set up security when running samples provided with the plugin. The plugins provide a default config.properties file that is used when running the sample projects. However, custom implementations that use SSL require that you generate corresponding custom certificates and keys and add them to the implementation's config.properties file. See Customizing a Java Card Platform if you are creating and deploying a custom Java Card 3 client-server implementation. Unless your custom SSL certificates and keys are issued by a valid certificate authority they will be treated by the client application as untrusted certificates. To successfully run your client-server application with your custom SSL certificates and keys, you must add your certificate to the browser's list of accepted certificates.

See the appropriate Java Card 3 platform documentation on the Java Card 3 Kenai project site for detailed information about setting up security for a project.
