C/C++ Project Properties Dialog Box: Linker

See Also 

The Linker (General) panel of the Project Properties dialog box let you specify linker options for your C or C++ Application or Dynamic Library project.

Output. The default output path is ${CND_DISTDIR}/${CND_CONF}/${CND_PLATFORM}/project-name, which places the output in a directory path that indicates the name of the project configuration and the tools and OS used to build. The binary output filename uses the lowercase name of the project. The output directory is located in the project directory by default. You can change the output directory and filename as you like.

Additional Library Directories. Type the absolute or relative path to the directories containing libraries for your project, or click the button to browse to the directories and add them. On the file browser dialog box, you can specify Absolute or Relative path.

Strip Symbols. Select the checkbox to remove symbol tables from the binary.

Additional Dependencies. Select the type of multithreading, if any, used by the project.

Tool. The command to run the linker.

Libraries. Click the field to add libraries to your project and specify binding options. See Creating Dependencies Between C or C++ Projects.

Additional Options. Type any other linker command line options in the text field, or click the button to open a dialog box to see the options that are currently in effect. In the dialog box, you can type more flags in the additional option in the Additional Options panel.
