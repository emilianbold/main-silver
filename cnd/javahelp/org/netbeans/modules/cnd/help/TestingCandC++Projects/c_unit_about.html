The IDE provides support for generating and executing unit tests based on its own simple test framework, the CUnit framework, and the CppUnit framework. The CUnit and CppUnit frameworks are open source products that support development of tests for C and C++ and provide harnesses for running the tests.

The IDE supports CUnit and CppUnit unit testing if the frameworks are separately installed on your system.

C/C++ Test Types in the IDE

You can use the IDE to create the following:

  • Simple C Tests.
  • Simple C++ Tests.
  • CUnit Tests.
  • CppUnit Tests.

You can generate tests by right-clicking the Test Files folder of a project in the Projects window and selecting New > New test-type.

Unit Test Structure

The IDE represents unit tests as subfolders of the Test Files folder. By default, when the IDE generates C/C++ tests each test file has the name of the class it is testing appended by the word Test (for example, MyClassTest.java).

Each standard project has a default test folder that is used to store the unit tests. This folder is displayed as the Test File node in the Projects window. You can add any number of test folders to your project. Test files and the source files they test cannot be located in the same source tree.???
