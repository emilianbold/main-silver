Inline Refactoring Window

See Also

The Inline Refactoring window provides a compact view of a source code compared to the proposed refactoring and simplifies navigation between different parts of code.

You can open the Inline Refactoring window by selecting a local variable (temp), method, or constant and choosing Refactor > Inline from either the main IDE's or popup menu. By default, the Inline Refactoring window opens in the lower right corner of the IDE.

The IDE provides the following inline refactorings:

  • Inline Temp. Replaces all references to a local variable with its initial value.
  • Inline Method. Replaces the current method calls with the method's body.
  • Inline Constant. Replaces a constant reference with its defined value, removing the constant declaration if there are no other references to it.
  • You should always perform a clean build after completing any refactoring commands. You can do a clean build by right-clicking the project's node in the Projects window and choosing Clean and Build Project.
