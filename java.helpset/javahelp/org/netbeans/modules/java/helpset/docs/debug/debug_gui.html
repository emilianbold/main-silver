Debugging Java GUI Applications

See Also

When debugging a GUI form, you can use the Visual Debugger to help you locate the form elements in your source code, set breakpoints and explore component listeners.

To use the Visual Debugger, you run the debugger and then take a GUI snapshot of the application UI. When you select a component in the GUI snapshot, the IDE will display the location of the component in the hierarchy tree of components in the Navigator window. The IDE will also display the properties of the selected component in the Properties window.

You can right-click a component in the GUI snapshot and select the following commands in the popup menu.

  • Go to Component Declaration. Select this to open the source file at the line where the component is declared.
  • Go to Component Source.Select this to open the source file at the line where the component is defined.
  • Go to Hierarchy Addition. Select this to open the source file where the component was added to the parent container.

    You need to enable this command in the Java Debugging tab in the Miscellaneous category in the Options window.

  • Show Listeners. Select this to open the Events window and view the order of events and the listeners associated with the selected component.
  • Toggle Component Breakpoint. Select this to set a breakpoint on the selected component. You can view, enable and disable the breakpoints in the Breakpoints window.

To take a GUI snapshot:

  1. Start a debugging session.
  2. Choose Debug > Take GUI Snapshot in the main menu.

The GUI snapshot will open in a tab in the main window.

After you set a breakpoint you will need to start a new debugging session to enable the breakpoint.
