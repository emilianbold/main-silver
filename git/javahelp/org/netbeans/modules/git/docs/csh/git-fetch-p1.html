Git Fetch from Remote Repository Wizard: Remote Repository

See Also

You use the Fetch from Remote Repository wizard to get the changes from the original remote repository that you do not have yet.

To open the Fetch from Remote Repository wizard, choose Team > Git > Remote > Fetch.

At the Repository page, you have to choose either:

  • Configured Repository. Select an earlier accessed Git root location used in the IDE from the drop-down list.
  • Specify Git Repository Location. Set the path to a remote Git repository that has not been accessed yet.
    • Remote Name. Specify the name of a remote Git repository.
    • Repository URL. Select a Git root protocol and location from the drop-down list. The drop-down list contains a list of protocols supported by the IDE.
    • User. Enter your user name for access to the repository. (Required for http and https protocols.)
    • Password. Enter your password for access to the repository. (Required for http and https protocols.)
    • Save Username and Password. Select the option if you want the IDE to remember your user name and password.
    • Proxy Configuration. Click the Proxy configuration button to specify details of your proxy configuration to connect to the Git repository.
