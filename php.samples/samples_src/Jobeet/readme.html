What is Jobeet?

Jobeet is PHP application built on Symfony framework. It was created by Symfony team and used in begineers tutorial. It's fully working imaginary application with frontend for visitors and backend for administrators with demonstrative usage of Doctrine ORM and sfFormExtraPlugin.

This sample demonstrates following features:

  • Symfony framework support in NetBeans
  • Symfony features such as
    • Doctrine ORM usage
    • Working frontend and backend application with sfFormExtraPlugin
    • Symphony i18N usage
    • And much more

Requirements

Follow these before you run the sample:
  • PHP server (Apache, for example)
  • PHP 5.2.x or higher
  • MySQL server
  • symfony 1.2.x (installation manual below)
  • PEAR (optional)

What is included?

  • Entire Symfony jobeet project as it is on svn.jobeet.org with documentation and without symfony sources

Symfony Installation

Ok. Stop for a moment and think. Either you just want to try symfony or you want to have symfony installed and develop real applications.

  1. Try only - continue with setup without installation
  2. For longer - continue with pear installation (slower but for whatever project) (if you prefer svn checkout read doc/03-Running-Symfony.txt)

Setting Up Without Installation

  1. Download symfony 1.2 from http://www.symfony-project.org/installation
  2. Extract
  3. Copy to lib/vendor/symfony. So the structure will be like

    lib/vendor/
    |-- Zend
    | | ... some Zend framework files for jobeet project
    `-- symfony
    |-- CHANGELOG
    |-- COPYRIGHT
    |-- LICENSE
    |-- README
    |-- UPGRADE_TO_1_1
    |-- UPGRADE_TO_1_2
    |-- data
    |-- doc
    |-- lib
    |-- licenses
    |-- package.xml.tmpl
    `-- test
  4. Symfony is not on your include path now so you need to provide path to symfony.bat on windows or symfony file on linux

    you can do so in menu Tools -> Options -> Php -> Symfony if you followed instructions above then symfony file will be in lib\vendor\symfony\data\bin\symfony.bat for Windows users lib/vendor/symfony/data/bin/symfony for Ubuntu users

    Symfony is done. Now continue with project setup.

Installing the symfony PEAR Package

  1. Make sure you have PEAR installed.

    The symfony PEAR package contains the symfony libraries and all its dependencies. It also contains a script that will extend your CLI to include the `symfony` command.
  2. The first step to install it is to add the symfony channel to PEAR, by issuing this command:

    pear channel-discover pear.symfony-project.com>
  3. To see the libraries available in this channel, type the following:

    pear remote-list -c symfony
  4. Now you are ready to install the latest stable version of symfony. Issue this command:

    pear install symfony/symfony-1.2.11

    downloading symfony-1.2.11.tgz ...
    Starting to download symfony-1.2.11.tgz (1,283,270 bytes)
    .................................................................
    .................................................................
    .............done: 1,283,270 bytes
    install ok: channel://pear.symfony-project.com/symfony-1.2.11
  5. That's it. The symfony files and CLI are installed. Check that the installation succeeded by calling the new `symfony` command line, asking for the version number:

    symfony -V
    symfony version 1.2.11 (/path/to/the/pear/symfony/lib/dir)

    The symfony libraries are now installed in directories as follows:

    * `$php_dir/symfony/` contains the main libraries.
    * `$data_dir/symfony/` contains the web assets used by symfony default modules.
    * `$doc_dir/symfony/` contains the documentation.
    * `$test_dir/symfony/` contains symfony core unit and functional tests.

    The `_dir` variables are part of your PEAR configuration. To see their values, type the following:

    pear config-show

    Symfony is done. Now, continue with project setup.

Project Setup

  1. change line 3 of config/ProjectConfiguration.class so that it will include sfCoreAutoload.class.php from your symfony folder in directory.

    If you just copied symfony to lib/vendor/ setup path to dirname(__FILE__).'/../lib/vendor/symfony/lib/autoloadsfCoreAutoload.class.php

    If you installed using PEAR then you should have symfony on include path:

    Your include path is listed in phpinfo() output.

    Linux users can usualy find out that include_path=".:/usr/share/php". There should be symfony folder. Your path is symfony/autoload/sfCoreAutoload.class.php then.

    Windows users are in more complicated situation. However you can find out where your include_path directives point from phpinfo() output or add symfony to include path. Example of php include path on Windows is include_path=".;C:\php5\pear"
  2. make sure Jobeet's cache folder is writable by everyone (or webserver full access at least)
  3. make sure Jobeet's log folder is writable by everyone (or webserver full access at least)
  4. setup database connection in config/databases.yml so that it will correspond to your database connection settings
  5. call symfony command (right click on project name -> Symfony -> Run Command)

    doctrine:build-all-load
  6. setup web server doc root to point to /web.
    Example:

    <VirtualHost *:80>


    ServerAdmin webmaster@localhost
    DocumentRoot /your_web_folder/Jobeet/web
    AccessFileName .htaccess


    <Directory /your_web_folder/Jobeet/web>
    Options Indexes FollowSymLinks MultiViews
    AllowOverride all
    Order allow,deny
    allow from all
    </Directory>


    </VirtualHost>
  7. Don't forget to change your_web_folder to correct path. Type http://localhost to your browser. If something is wrong then you can try http://localhost/frontend_dev.php that should have debugging output. You can set up your project to called from diferrent url, of course, such as http://localhost/var/www/Jobeet/web and it should work
  8. invoke symfony commands

    guard:create-user admin admin
    guard:promote admin

    this will create user admin/admin and make him/her super administrator. You need this credentials to enter backend application http://localhost/backend.php

Notice

Check if you have path to php interpreter set correctly if you don't see Symfony commands. If you are using wamp/xampp or similar product then you need to provide path to php.exe or php (depends on platform) in Tools -> Options -> PHP.
