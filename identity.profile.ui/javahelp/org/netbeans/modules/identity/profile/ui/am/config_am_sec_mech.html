Configuring Security Mechanism Profiles for Sun Java System Access Manager Instances

See Also

For each security mechanism, the IDE includes a predefined profile for that mechanism. You can modify these predefined profiles to suit your implementation.

  • The configuration changes you make in the Edit security-mechanism Profile dialog box are applied in real-time to the server instance.

To configure a security mechanism profile for a Sun Java System Access Manager instance:

  1. In the Services window, expand the the Sun Java System Access Managers node.
  2. Expand the server-instance node for which you want to configure a security mechanism profile and then expand the Profiles node.
    A subnode appears for each security mechanism profile that you can configure.
  3. Right-click the security-mechanism-profile node and choose Edit.
    The Edit security-mechanism Profile dialog box opens.
  4. Select the Sign Response checkbox.
    • If you clear this checkbox for the UserNameToken profile, the settings under Existing Certificate Settings are disabled.
  5. Under Existing Certificate Settings, to use a custom keystore, clear the Use Default Key Store checkbox and enter the required information in each of the following fields:
    • Keystore Location. Specifies the location of the keystore file that contains the key alias. This should be specified as an absolute path.
      • The Browse button for the Keystore Location field is only enabled for local Access Manager instances.
    • Keystore Password. Specifies the password needed to access the keystore file.
    • Key Alias. Specifies the key alias for the entry in the keystore. Each entry in a keystore file is identified by an alias string.
    • Key Alias Password. Specifies the password needed to access the key alias for the entry in the keystore.
    • If you are using the Liberty security mechanism, you must perform the steps described in Using a Custom Keystore for the Liberty Token Profile to prevent failure at runtime.
  6. For the UserNameToken profile, create an entry for each web service client by doing the following:
    1. Click Add.
    2. In the Add User dialog box, type the user name and password for the web service client.
      The user name must be unique.
    3. Click OK.
  7. At the Edit security-mechanism-profile Profile dialog box, click OK.
