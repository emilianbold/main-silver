Attach Profiler: Attach Settings Dialog Box

See Also

The Attach Wizard helps you configure the settings for profiling an application using the attach method. Attach settings are associated with the project.

The Attach Settings dialog box enables you to configure and review the settings for attaching the profiling tool to the application. If the settings are correct, do one of the following:

To attach the profiler to an application you first need to specify location of the target JVM by selecting one of the following Target options and then performing the steps to

  • Local. Select this option if the application is running on the local machine.

    If you select this option you need to select one of the following Connection options.

    • Dynamic. Connect to a process that is running.
    • Direct. Select this option if you want the application to start after the profiler attaches. If you select this option you will need to modify the startup script of the application for standalone Java SE applications. If you are attaching to a server you will need to modify the startup script and configuration file of the server.
  • Remote. Select this option if the application is running on a remote machine.

    If you select Remote you need to specify the following connection details for the remote host.

    • Hostname. Specify the hostname of the target.
    • OS & JVM. Select the operating system of the target.

    After you specify the connection details you need to perform the steps in the dialog box. You will need to start the remote application on a Remote Profiling Pack that the IDE will generate based on the connection details that you specified.
