OQL Console: Save Query Dialog Box

See Also

The Save Query dialog box enables you to save an OQL query and assign a name and description to the query. Saved queries are listed in the Saved Queries pane of the OQL Console tab in the Heap window.

You open the Save Query dialog box by clicking Save in the Query Editor pane of the OQL Console tab in the Heap window.

When you save a query you can choose from the following options.

  • Create a New Query. Select this option to save the query in the Query Editor pane as a new named query and enter the following details about the query.
    • Name. Type a name to identify the query.
    • Description. (Optional) Type a short description of the query.
  • Use Existing Query. Select this option to modify an existing custom saved query. When you select this option you need to select a custom query from the list. The query in the Query Editor pane will replace the query associated with the custom query that you select.

    This option is only available if custom queries exist.

Saved queries are added to the list under the Custom node in the Saved Queries pane. You can open and run the saved queries against any Java heap snapshot. You can delete saved queries in the Saved Queries pane.
